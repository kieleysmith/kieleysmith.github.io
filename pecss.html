Introduction and Definition of Trend

Progressive Enhancement is a web design strategy in which a document is designed with the functionality of older software and browsers in mind. The document is then enhanced through external links such as Cascading Style Sheets. This is a reversal of the graceful degradation mindset in which a document was created incorporating the latest and greatest trends for newer browsers while hopefully maintaining some sort of basic functionality (1). This was shortsighted in practice however, due to the rise of mobile devices.

CSS3 aids Progressive Enhancement strategy in that basic functionality is preserved despite differences in browsers. It provides workarounds that cut down on code and the need for java-scripting in certain circumstances (2), such as text box corner design, multiple background images, and custom fonts. Furthermore, CSS3 provides extensions to increase functionality across various browsers. Even where not fully supported, CSS3 ensures that a page is still responsive at the most basic of levels.

Tutorial/Examples

Progressive Enhancement functionality with CSS3 and browser extensions is perhaps best demonstrated by creating rounded borders around an element. The following code:

nav.horizontal li a { background-color: red; color: white; display: block; text-decoration: none; -moz-border-radius: 30px/25px; -webkit-border-radius: 30px/25px; border-radius: 30px/25px; }

creates rounded corners and takes into consideration various browser software. Where not supported, CSS3 will simply display the element with square corners.

Trend Support

Although CSS3 is largely supported by a wide range of browsers, individual CSS3 properties are not always fully supported. Properties can be dependent on both browser type and browser software versions.

W3C provides a current list of cross browser functionality.

Naturally, future support will be browser dependent.
