Introduction and Definition of Trend

Historically, HTML has been a very static language - as is its name ("markup language"). However, as the years have gone by, a need for more dynamic content arose. At first, this need was filled by other languages embeded into the webpage such as Java and Flash applets. Javascript was developed as a means for web developers to make their websites more dynamic without having to embed other languages into the browser using plugins - the browser itself would interpret the language. It didn't see major adoption, however, until the late 00's, and it still couldn't achieve the level of sophistication that Flash (then the most common web scripting language) or Java could, only being able to use simple interactivity with the user. HTML 5, however, set out to change that with numerous changes to Javascript/HTML integration and the canvas element.
The canvas element introduced an entirely new form of interaction with the user - in-browser animations, video games, and complex rendering were all now achievable and faster than any of the competitors. Nowadays, Javascript and the canvas element tower above their competitors, as Oracle gets rid of its applet system entirely and the removal of NPAPI makes Flash struggle.
